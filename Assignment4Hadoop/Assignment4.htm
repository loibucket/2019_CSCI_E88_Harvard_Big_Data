    Assignment 4 [1 week] Hadoop MR

    Total points: 100

    NOTE: please use the following submission template for you solution document: Assignment4_SubmissionTemplate.docx

    For problems 1-3: use the same input logs as for Assignment 2/3 and place them into some AWS S3 bucket

    NOTE: we would like you to execute all your MR jobs on a real Hadoop (EMR) cluster; you can still opt to run them locally - in which case you will lose 10 points per each problem not run on an EMR cluster.

     

    Problem 1: [points: 15]

    • Create an S3 bucket and place the 4 input files there
    • Run the bundled example 'wordcount' program  to verify that your cluster is functioning OK 
      • Tip:  you will want to use a command similar to:
      • 'hadoop jar /usr/lib/hadoop-mapreduce/hadoop-mapreduce-examples.jar wordcount <your_input> <your_output>'
    • Your inputs have to be sourced from S3 bucket and outputs saved in S3 bucket
    • Examine the job logs or console output  and show how many splits, map tasks and reduce tasks your job has had.
    • Show job results

     

    Problem 2: unique counts [points: 40]

    • write a Hadoop MapReduce job that processes all 4 input files form the S3 bucket and computes answers to the following queries:
    • Query1: get count of unique URLs per hour
    • Query2: get count of unique visitors per URL per hour
    • Query3: get count of unique (by UUID) events/clicks per URL per hour (one unique click == one log line in our test log files, since we assume they all have unique UUIDs)
    • Tip: we suggest you create a separate MR job for each query; MR jobs should be executed on the AWS EMR cluster
    • show results of each query for the required keys

     

    Problem 3: time range queries [points: 30]

    Modify your Problem 2 program to calculate results for the following Query4:

    • get count of unique URLs by country by hour for a  specified time range [t1-t2]
      • where t1 = 09/13/2019 5PM UTC and
      • t2 = 09/14/2019 9AM UTC
      • Both range ends are inclusive
      • Show results for the specified keys

     

    Problem 4: [points: 15] Explore MR Job configurations

    • research how to specify number of reduce tasks for your jobs
      • hint: consider "-Dmapreduce.job.reduces=xx" setting
      • ref: https://hadoop.apache.org/docs/current/hadoop-streaming/HadoopStreaming.html#Specifying_the_Number_of_Reducers
    • run any of the jobs (Problem 1, 2 or 3) with two settings for the number of reduce jobs: 2 and 4
    • compare the results - what is the difference? Explain
    • Explore execution of your jobs in the EMR ResourceManager UI (see below) - show statistics for each of your jobs - any differences?

     

    Problem 5: Bonus: top N queries [points: 15]

    Modify your Problem 2 or 3 program to calculate results for the following Query5:

    • get top 5 URLs by avg daily TTFB, per each day in the input data set; consider "top" URLs to be those that have the smallest avg TTFB
