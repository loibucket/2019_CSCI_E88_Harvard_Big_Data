    Assignment 5 [1 week]  Spark Batch processing and Parquet

    WARNING: Please do not start working on this assignment on the last day before the deadline! :-) This is especially important if you've never worked with Spark before. While not particularly difficult to work with, Spark setup and dependencies might be challenging in some cases. The sooner you start working on the assignment - the sooner you can start asking questions and have a much better chance to get a timely help from TAs and/or your classmates. The last days of the week are usually very busy for the teaching staff as we are working on the next Lab and homework ...

     

    Please use the following Submission Template: Assignment5_SubmissionTemplate.docx

    For Problems 1-3: use the same CSV files in S3 you used for HW4 as your input.

    NOTE: we would like you to execute all your Spark jobs on a real Spark EMR cluster; you can still opt to run them locally - in which case you will lose 10 points per each problem not run on an EMR cluster.

     

    Problem 1: unique counts [points: 40]

    • Use Spark RDDs only for this problem, do not use other API’s
    • write Spark jobs that process all 4 input files from the S3 bucket and compute answers to the following queries:
      • Query1: get count of unique URLs per hour
      • Query2: get count of unique visitors per URL per hour
      • Query3: get count of unique (by UUID) events/clicks per URL per hour (one unique click == one log line in our test log files, since we assume they all have unique UUIDs)
    • Tip: we suggest you create a separate Spark job for each query
    • examine Spark Admin Web UI (or relevant logs) and show execution stages and input/output sizes for each job
    • show results of each query for the required keys (see submission template for required keys - same as in HW4)

     

    Problem 2: Spark with Parquet [points: 20]

    • Write a Spark job that reads in all your input CSV files and writes them out in the Parquet format
    • Either S3 or HDFS is ok for the output files location
    • Add compression to Parquet output files
    • examine structure of one of the generated Parquet files using the parquet-tools utility, and specifically - using the 'meta' command. How many Row Groups your file has? Do you see a compression type in the meta info?
      • Ref: https://github.com/apache/parquet-mr/tree/master/parquet-tools
      • Tip: to run it on EMR using 'hadoop jar parquet-tools-<x.x>.jar meta s3://<your_bucket/yourfile.parquet>' you may need to download (https://mvnrepository.com/artifact/org.apache.parquet/parquet-tools) and scp the parquet-tools jar to your EMR master node

     

    Problem 3: time range queries [points: 20]

    Write a Spark job that:

    • uses generated Parquet input files and calculates results for Query4:
    • get count of unique URLs by country by hour for a  specified time range [t1-t2]
      • where t1 = 09/13/2019 5PM UTC and
      • t2 = 09/14/2019 9AM UTC
      • Both range ends are inclusive
      • Show results for the specified keys (see submission template for keys, same as in HW4)
    • Enable query execution plan via explain() command
    • Examine the execution plan and verify whether you were taking advantage of the columnar-based format or not

     

    Problem 4: SparkSQL [points: 20]

    Write a Spark job that:

    • reads in generated Parquet input files
    • Implements slightly modified Query 3 using Spark SQL:
      • Query3: get count of events  per URL for a specified hour: 2019-09-12:14
    • Enable query execution plan
    • Examine the execution plan and verify whether you were taking advantage of the columnar-based format or not

     

    Problem B1: Bonus: top N queries [points: 15]

    Write a Spark job (either Core RDD or SparkSQL can be used) that:

    • uses generated Parquet input files and calculates results for Query5:
    • get top 5 URLs by avg daily TTFB, per each day in the input data set; consider "top" URLs to be those that have the smallest avg TTFB
